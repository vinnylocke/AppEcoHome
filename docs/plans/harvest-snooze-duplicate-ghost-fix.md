# Plan — Fix snoozed harvest task duplicate-key error + reappearing overdue

## Bug

Reported by Vinny: snoozing a harvest task hides it, but the SAME ghost gets re-generated by the engine. Snoozing again calls `materializeTask`, which tries to INSERT a row with the same `(blueprint_id, due_date)` key — the unique constraint rejects it. The hidden materialised task also reappears as overdue once `next_check_at` passes, even though the user has another task at the same date.

## Root cause

[src/lib/taskEngine.ts:292-300](src/lib/taskEngine.ts#L292-L300) — my Wave-20 `next_check_at` filter is applied INSIDE `rawTasks`. Both ghost branches ([line 337-340](src/lib/taskEngine.ts#L337-L340) for harvest, [line 386-389](src/lib/taskEngine.ts#L386-L389) for normal) then use `rawTasks.some(...)` to decide whether a ghost should be suppressed. Snoozed tasks are missing from `rawTasks` → suppression misses them → ghost regenerates → duplicate.

## Fix

Build a separate `materialisedKeys: Set<string>` from the **unfiltered** physical-tasks result. Use it for ghost suppression. Keep the existing `rawTasks` filter for what gets returned to the client.

```ts
const physicalRows = physicalTasks || [];
const materialisedKeys = new Set(
  physicalRows
    .filter((t: any) => t.blueprint_id && t.due_date)
    .map((t: any) => `${t.blueprint_id}:${t.due_date}`),
);
const rawTasks = physicalRows.filter(/* visibility filters as today */);
// Ghost branches: replace rawTasks.some(...) with materialisedKeys.has(`${bp.id}:${date}`)
```
