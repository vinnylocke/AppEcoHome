## Batch 1 ‚Äî Security / paid-feature bypass (CRITICAL)

### 1.1 `[x]` agent-chat has no server-side AI gate or rate limit
- **Where:** `supabase/functions/agent-chat/index.ts` (only `requireAuth`).
- **Why it's a bug:** RHO-10 gated Plant Doctor Chat client-side only; the App.tsx comment claims "Server re-verifies ai_enabled on every chat call" ‚Äî it doesn't. A Sprout user can invoke the function directly and get unlimited Gemini chat (paid Sage feature, unbounded cost).
- **Fix (as landed):** `guardAiByUser` added after `requireAuth`, applied to all four chat actions. `enforceRateLimit` was NOT added ‚Äî the function already rate-limits via the per-tier daily message quota (`check_ai_message_quota` RPC), which the gate now sits in front of.
- **Docs:** `17-tier-gating.md`, `05-tools/03-plant-doctor-chat.md`.

### 1.2 `[x]` plant-doctor AI gate skipped when client omits `homeId`
- **Where:** `supabase/functions/plant-doctor/index.ts:653-660` ‚Äî `if (homeId && !skipAiGate)`.
- **Why:** `homeId` is client-controlled and optional for `diagnose` / `identify_pest`; omit it and no tier gate runs.
- **Fix:** when `!homeId`, fall back to `guardAiByUser(supabase, userId)` for non-skip actions.
- **Docs:** `17-tier-gating.md`, `05-tools/02-plant-doctor.md`.

### 1.3 `[x]` "Editor" invites cannot be redeemed ‚Äî role vocabulary mismatch
- **Where:** `home_invite_tokens` CHECK allows `owner/editor/viewer`; `home_members` CHECK allows `owner/admin/member/viewer`; `redeem-home-invite/index.ts:135-141` inserts `invite.role` verbatim ‚Üí constraint violation ‚Üí 500. `src/lib/permissions.ts` has no `editor` role either.
- **Why:** two migrations introduced different role vocabularies; the default invite role ("editor") hits the mismatch, so the feature's default path is broken end-to-end.
- **Fix:** map `editor ‚Üí member` at redemption (smallest safe fix; keeps invite-facing vocabulary). Follow-up: align docs (`02-members-permissions.md` documents owner/editor/viewer).
- **Docs:** `07-management/02-members-permissions.md`, `19-rls-patterns.md`.

## Batch 2 ‚Äî Dashboard stats & recent-regression fixes (HIGH)

## Batch 3 ‚Äî Task engine / scheduling correctness (HIGH)

### 3.1 `[x]` Ghost dates shift ‚àí1 day in UTC-negative timezones
- **Where:** `src/lib/taskEngine.ts:436-484` ‚Äî `new Date("YYYY-MM-DD")` (UTC midnight) formatted with local getters (`getLocalDateString`). Also `collectHarvestWindowDates` (:81-90).
- **Why:** every ghost lands a day early for Americas users; the dedup keys then miss cron-materialised real tasks ‚Üí duplicates; materialising inserts wrong-date rows.
- **Fix:** do occurrence math in pure UTC (`Date.UTC` + `toISOString().slice(0,10)`) or parse with local `new Date(y, m-1, d)` to match the formatter. Unit tests with UTC-negative tz simulation.

### 3.2 `[x]` generate-tasks cron materialises archived + paused blueprints
- **Where:** `supabase/functions/generate-tasks/index.ts:35-38` ‚Äî no `is_archived` filter, no `paused_until` handling; contradicts `04-data-model-tasks.md`.
- **Fix:** `.eq("is_archived", false)` + skip while `paused_until > today`.

### 3.3 `[x]` Cron and ghost engine use different recurrence anchors ‚Üí duplicate tasks
- **Where:** `generate-tasks/index.ts:97-108` anchors on last task / clamps to today; `taskEngine.ts:439-451` projects strictly from `start_date`.
- **Fix:** make the cron project from the `start_date` grid (`start + ceil((today-start)/freq)*freq`), ignoring off-grid rows.

### 3.4 `[x]` Paused blueprint suppresses ALL ghosts (incl. post-pause) and resurrects pause-period ghosts later
- **Where:** `taskEngine.ts:382-385` ‚Äî `if (isPaused) return;`.
- **Fix:** skip individual `ghostDateStr < bp.paused_until` (string compare), permanently.

### 3.5 `[x]` Dashboard today-counts re-implement recurrence with different rules
- **Where:** `src/App.tsx:763-810` ‚Äî counts archived/paused blueprints, misses Skipped rows, miscounts harvest windows.
- **Fix:** filter archived/paused, include Skipped in the materialised set, special-case harvest windows.

### 3.6 `[x]` `completed_at` UTC slice vs local date ‚Äî ticks on wrong day
- **Where:** `TaskCalendar.tsx:242-296`, `taskEngine.ts:329-333`.
- **Fix:** `getLocalDateString(new Date(t.completed_at))` (correct here ‚Äî real timestamp).

### 3.7 `[x]` Shift-blueprint off-by-one (UTC-negative tz)
- **Where:** `src/components/TaskList.tsx:723-729` ‚Äî same UTC-parse/local-format mix.
- **Fix:** same treatment as 3.1.

### 3.8 `[x]` Recurrence preview labels one day early
- **Where:** `src/lib/scheduleDatePreview.ts:82-88`.
- **Fix:** add `timeZone: "UTC"` to the `Intl.DateTimeFormat` options.

### 3.9 `[x]` Daily Brief frost hint keyed on UTC date
- **Where:** `src/components/DailyBriefCard.tsx:70`.
- **Fix:** use `getLocalDateString(now)`.
