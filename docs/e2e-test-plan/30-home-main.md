Covers the default `/dashboard` view (`HomeMain`, labelled "Dashboard") shipped in Phases 1‚Äì2 of `docs/plans/new-home-dashboard.md` and merged with the old Overview surface in design-overhaul **Phase 4.2**: the **4-tab switcher** (Dashboard / Locations / Calendar / Weather ‚Äî the Overview tab is gone), the legacy `?view=dashboard` **and** `?view=overview` ‚Üí home fallthrough (neither param is in the App.tsx view whitelist any more), the garden overview grid, persona-default quick actions, the density‚Üí**posture** toggle (localStorage `rhozly:home:preset` since Stage 4, with the legacy `rhozly:home:density` mirrored; the Workbench posture switches the ONE hero to its console voice, adds the Attention inbox, and puts the full task board behind "Open board ‚Üí" on the Calendar ‚Äî the Stage-4 note below is authoritative), the compact today's-tasks section, and (Phase 2) the `home-overview` telemetry ‚Äî sensor/valve chips on area rows plus the ranked attention row. The relocated Overview content itself is exercised by the classic-dashboard specs in [05-dashboard.md](./05-dashboard.md).

**Note:** `DashboardPage.goto()` now seeds `localStorage["rhozly:home:density"] = "detailed"` via an init script and navigates to plain `/dashboard` ‚Äî the classic-dashboard specs (Section 5) ride on the home's Detailed density instead of a separate Overview URL.
