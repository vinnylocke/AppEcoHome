# 30. Home (Main Dashboard)

**Spec file:** `tests/e2e/specs/home-main.spec.ts`
**Page Object:** `tests/e2e/pages/HomeMainPage.ts`
**Seed dependencies:** `00_bootstrap.sql` (user, home), `01_locations_areas.sql` (Outside Garden + Indoor Space, 5 areas — HOME-002 asserts "Raised Bed A" / "South Border" rows and the per-worker location UUIDs `0000000N-0000-0000-0001-00000000000{1,2}`), `02_plants_shed.sql` (6 plants/inventory items for the growth-state dots), `03_tasks_blueprints.sql` (today's + overdue tasks for the status-strip chips and compact task list), `13_integrations.sql` (ecowitt integration + soil sensor on Raised Bed A with a fresh reading + water valve on South Border — real-data backing for the Phase 2 telemetry chips; HOME-008 itself mocks `home-overview`, so the seed keeps the unmocked page realistic rather than being asserted directly)
**App-reference:** [02-dashboard/17-home-main.md](../app-reference/02-dashboard/17-home-main.md)

Covers the default `/dashboard` view (`HomeMain`, labelled "Dashboard") shipped in Phases 1–2 of `docs/plans/new-home-dashboard.md` and merged with the old Overview surface in design-overhaul **Phase 4.2**: the **4-tab switcher** (Dashboard / Locations / Calendar / Weather — the Overview tab is gone), the legacy `?view=dashboard` **and** `?view=overview` → home fallthrough (neither param is in the App.tsx view whitelist any more), the garden overview grid, persona-default quick actions, the density toggle (localStorage `rhozly:home:density`; Simple vs Detailed — since redesign Stage 2 Detailed switches the ONE hero to its console voice and adds the full TaskList and Garden Snapshot; the AI cards + Week Ahead render in both densities), the compact today's-tasks section, and (Phase 2) the `home-overview` telemetry — sensor/valve chips on area rows plus the ranked attention row. The relocated Overview content itself is exercised by the classic-dashboard specs in [05-dashboard.md](./05-dashboard.md).

**Note:** `DashboardPage.goto()` now seeds `localStorage["rhozly:home:density"] = "detailed"` via an init script and navigates to plain `/dashboard` — the classic-dashboard specs (Section 5) ride on the home's Detailed density instead of a separate Overview URL.

> **Home redesign Stage 1 (2026-07-20, docs/plans/home-redesign-two-postures.md):** `HomeStatusStrip` is now the display-scale **sentence hero** — the old chip row died with its testids (`home-strip-weather/tasks/tasks-headline/pending/skipped/postponed/overdue/frost`; no spec referenced them). New hero selectors: `hero-sentence`, `hero-plan-day`, `hero-weather-chip` (sentence voice) and `hero-console-line`, `hero-seg-{id}` (console voice, consumed from Stage 2). The wrapper keeps `home-status-strip` (dashboard_tour step 2). Chrome: the sync pill (`dashboard-sync-status`) renders **only when stale >5 min**, the static pull-to-refresh hint is deleted, the 4-tab switcher is a compact pill row (same testid/roles), and the single-slot promo cascade renders **below the hero** via HomeMain's `promoSlot` prop. `DashboardPage.calendarPrevButton/NextButton` were repointed from page-wide `.lucide-chevron-*` class locators (went strict-mode ambiguous with the hero's chevron) to the calendar buttons' own aria-labels. Re-verified 2026-07-20: home-main 8/8, layout 12/12.
>
> **Home redesign Stage 2 (2026-07-20 — the cuts and moves):** ONE hero now serves BOTH densities — `DailyBriefCard` is **deleted**; Detailed mounts the **console voice** (`hero-console-line` + `hero-seg-{tasks|overdue|weather|frost|sun}`, each segment deep-linking, zero-value segments dropped) plus the **migrated ask-AI chip** (SAME testid `daily-brief-ask-ai`, same RHO-11 `aiEnabled` gate). `daily-brief-card` is now **uniquely** GardenBrainBriefCard's testid (the collision died with the deletion; a Stage-3 rename is pending — HOME-012's selector is unambiguous meanwhile). The **Garden Walk tile now renders INSIDE `home-quick-actions`** as its featured full-width first tile (same `dash-garden-walk` testid + `state:{from:"/dashboard"}` contract; still gated on ≥ 5 plants) — HomeMain's standalone walk block is gone. The dashboard's **AttentionRow suppresses `overdue_tasks` + `weather_alert`** via the new `excludeKinds` prop (the hero owns overdue; the global WeatherAlertBanner owns alerts) — expect only `home-attention-{automation_failed|low_battery|soil_dry|harvest_closing}` on this page. **GardenSnapshot starts COLLAPSED for everyone** (the persona-follows-open effect is gone; expand via `dash-snapshot-toggle`; explicit toggles still persist via `rhozly:dashboard:snapshot-open`). `WeekAheadPreview`'s `describeWeekChips` dropped the task-count + weather-alert chips (the card leads with sow/harvest/prune windows). Re-verified 2026-07-20: unit 1543 tests / 142 files, home-main 8/8, targeted dashboard subsets 11/11 + 15/15.
