# 30. Home (Main Dashboard)

**Spec file:** `tests/e2e/specs/home-main.spec.ts`
**Page Object:** `tests/e2e/pages/HomeMainPage.ts`
**Seed dependencies:** `00_bootstrap.sql` (user, home), `01_locations_areas.sql` (Outside Garden + Indoor Space, 5 areas — HOME-002 asserts "Raised Bed A" / "South Border" rows and the per-worker location UUIDs `0000000N-0000-0000-0001-00000000000{1,2}`), `02_plants_shed.sql` (6 plants/inventory items for the growth-state dots), `03_tasks_blueprints.sql` (today's + overdue tasks for the status-strip chips and compact task list), `13_integrations.sql` (ecowitt integration + soil sensor on Raised Bed A with a fresh reading + water valve on South Border — real-data backing for the Phase 2 telemetry chips; HOME-008 itself mocks `home-overview`, so the seed keeps the unmocked page realistic rather than being asserted directly)
**App-reference:** [02-dashboard/17-home-main.md](../app-reference/02-dashboard/17-home-main.md)

Covers the default `/dashboard` view (`HomeMain`, labelled "Dashboard") shipped in Phases 1–2 of `docs/plans/new-home-dashboard.md` and merged with the old Overview surface in design-overhaul **Phase 4.2**: the **4-tab switcher** (Dashboard / Locations / Calendar / Weather — the Overview tab is gone), the legacy `?view=dashboard` **and** `?view=overview` → home fallthrough (neither param is in the App.tsx view whitelist any more), the garden overview grid, persona-default quick actions, the density toggle (localStorage `rhozly:home:density`; Simple vs Detailed — Detailed swaps the status-strip hero for the Daily Brief and adds the AI cards, full TaskList, Week Ahead and Garden Snapshot), the compact today's-tasks section, and (Phase 2) the `home-overview` telemetry — sensor/valve chips on area rows plus the ranked attention row. The relocated Overview content itself is exercised by the classic-dashboard specs in [05-dashboard.md](./05-dashboard.md).

**Note:** `DashboardPage.goto()` now seeds `localStorage["rhozly:home:density"] = "detailed"` via an init script and navigates to plain `/dashboard` — the classic-dashboard specs (Section 5) ride on the home's Detailed density instead of a separate Overview URL.

Key selectors: `home-main`, `dashboard-view-switcher`, `home-status-strip`, `home-overview-grid`, `home-location-card-<uuid>`, `home-area-row-<kebab-name>`, `home-quick-actions`, `home-quick-tile-<id>`, `home-density-simple`/`home-density-detailed`, `home-todays-tasks`, `home-tasks-see-all`, `home-empty-garden`, `home-sensor-chip`, `home-valve-chip`, `home-area-tasks-chip`, `home-attention-row`, `home-attention-<kind>` (`home-week-pulse` is gone — WeekPulse was deleted in Phase 4.2; its stacked-dot language now lives in the Garden Snapshot day strip).
