# Companion Plants Tab

> Shown inside the Instance Edit Modal and the Plant Detail Modal. Lists a plant's beneficial / harmful / neutral companions, generated by the `companion-planting` edge function (Verdantly data for Verdantly plants, AI otherwise) and cached server-side. Each companion has an ⓘ inline peek (info pills + description) and opens its own full care guide on tap, and any companion can be ticked and added to the Shed.

**Source file:** `src/components/CompanionPlantsTab.tsx` (~640 lines)

---

## Quick Summary

For the current plant the tab fetches companions via the cached
`companion-planting` edge function and renders three collapsible sections:

- **Beneficial** — plant-together companions, each with a one-line reason.
- **Harmful** — avoid-near antagonists, each with a reason.
- **Neutral** — no strong effect (collapsed by default).

Each companion row mirrors the shared plant-search experience: tap **ⓘ** for an
inline peek (sun / water / edible / toxic / wildlife pills + description), or
tap the **name** to open that companion's full care guide (Care / Grow Guide /
Companions / Light) in `PlantDetailModal`. Tick the checkbox on any companions
and "Add N to Shed" runs them through the source picker → bulk-add flow.

## Role 1 — Technical Reference

### Component graph

```
CompanionPlantsTab
├── Loading state (spinner)
├── Error states — ai_required (tier gate) | rate_limited | fetch_failed (retry + reason)
├── Empty state ("No companion plants for X — that's normal for carnivorous, aquatic, indoor-only species.")
├── Intro card ("What is companion planting?")
├── CompanionSection × 3 (Beneficial / Harmful / Neutral)
│   └── row: checkbox · name button (→ care guide) · ⓘ toggle
│       └── expanded: reason + PlantInfoPanel pills | loading | "open full care" link
├── Sticky "Add N to Shed" footer (when ≥1 checked)
├── PlantSourcePicker (choose per-plant source: api / ai / verdantly)
├── BulkSearchModal (review + bulk add)
└── PlantDetailModal (lazy — companion's own full care guide)
```

`PlantDetailModal` is imported via `lazy()` + `<Suspense>` to break the
`CompanionPlantsTab ⇄ PlantDetailModal` import cycle (the modal's own
Companions tab renders this component).

### Props

| Prop | Type | Source | Purpose |
|------|------|--------|---------|
| `source` | `string` | host | Plant source — decides Verdantly vs AI companion generation |
| `verdantlyId` | `string \| null` | host | Verdantly id when `source === "verdantly"` |
| `plantName` | `string` | host | The plant whose companions to generate |
| `homeId` | `string` | host | Shed inserts + provider/AI calls |
| `aiEnabled` | `boolean` | host | Sage+ AI gate (companion generation + AI care guide) |
| `isPremium` | `boolean` | host | Gates the add-to-Shed source picker options |
| `onPlantsAdded` | `() => void` | host | Fired after a bulk add so the host can refresh |

### Data flow — read paths

```ts
// 1. Companions — cached edge-function call (shared promise cache + server cache)
fetchCompanions({ source, verdantlyId, plantName, aiEnabled })   // src/lib/companionCache.ts
//   → supabase.functions.invoke("companion-planting", …)

// 2. ⓘ pills + care-guide clone source — via the shared resolver
//    src/lib/plantInfoResolver.ts (library-first → provider DB → AI last):
resolvePlantInfo(plant.name, plant.scientificName)    // { details, result }
//   library (plant_library) → libraryRowToPlantDetails → clone via plant_library_id
//   miss → Verdantly/Perenual → getProviderPlantDetails → clone from provider
//   miss → AI-by-name; Gemini only runs if the full care guide is opened
// `resolveCompanion` wraps this with its per-row details/result cache.
// The same resolver powers Plant Doctor Multi-ID's per-plant ⓘ + add-to-Shed.
```

### Data flow — write paths

- **Add to Shed** — ticked companions → `PlantSourcePicker` → `BulkSearchModal` →
  `plants` insert + auto seasonal `plant_schedules` (via `buildAutoSeasonalSchedules`);
  Verdantly harvest metadata adds a "Check for harvest" schedule.

### Edge functions invoked

| Function | Purpose |
|----------|---------|
| `companion-planting` | Generates beneficial/harmful/neutral lists (Verdantly or Gemini); reads/writes `companion_cache` |
| `image-proxy` | Persists a companion's image on add-to-Shed |
| `plant-image-search` | Gallery thumbnails inside the ⓘ `PlantInfoPanel` |
| `add-plant-to-library` (indirect) | Only if the user later AI-creates a missing companion in the library |

### Caching

- **Client:** `src/lib/companionCache.ts` — in-memory promise cache keyed by request, so a "pre-warm" call and the mount fetch share one network request; `invalidateCompanions` clears a stale entry before the one-shot auto-retry.
- **Server:** `companion_cache` table — AI results cached permanently, Verdantly results for 30 days (`VERDANTLY_TTL_MS`); empties are **not** cached. A cache hit returns without consuming the rate limit.
- **ⓘ details** are cached per row in component state (`companionDetails` + `companionResult`).

### Cron / scheduled jobs

None.

### Realtime channels

None.

### Tier gating

- **AI companion generation** (non-Verdantly plants) requires `aiEnabled` (Sage+) — otherwise the tab shows the `ai_required` gate. Verdantly plants generate from Verdantly data for everyone.
- **Rate limit** (`supabase/functions/_shared/rateLimit.ts`): companion-planting = Botanist 20 / Sage 40 / Evergreen 80 per window. Cache hits don't count; a 429 shows the dedicated `rate_limited` message (no auto-retry).
- The ⓘ provider fallback uses Verdantly (free for all) preferentially; Perenual self-gates inside `getEnabledProviders`.

### Beta gating

None.

### Permissions

Add-to-Shed inserts respect the home's RLS (member of `home_members`).

### Error states

| State | Result |
|-------|--------|
| `ai_required` | Tier-gate panel (Sage+ to generate AI companions) |
| `rate_limited` | "You've hit the limit — try again shortly" (no retry) |
| `fetch_failed` | Retry banner. The captured upstream message (e.g. `"Gemini gemini-2.0-flash returned no usable text (finishReason: SAFETY)"`) is surfaced below the headline via `data-testid="companion-error-detail"` so the user can see WHY it failed instead of an opaque Retry. One silent auto-retry was already attempted before this banner shows. |
| No companions | `data-testid="companion-empty-state"` — "No companion plants for {plantName}" with a sub-line explaining unusual species (carnivorous bog plants, strictly aquatic species, indoor-only houseplants) typically have no traditional companions. The edge function's prompt explicitly allows the AI to return empty arrays for those cases instead of inventing companions. |
| ⓘ library + provider miss | Reason only + "Open … for its full care guide →" link |

### Performance

- One companion call per plant viewed (shared cache + server cache).
- First call after navigation can fail transiently (cold start / token) → one silent auto-retry after ~900 ms before surfacing an error.
- ⓘ peek for a library miss costs one provider search + one details fetch, on demand, cached per row, behind the "Loading details…" spinner.

### Linked storage buckets

`plant-images` (via `image-proxy` on add-to-Shed).

## Role 2 — Expert Gardener's Guide

### Why open this tab

Companion planting is one of the oldest gardening tactics — pairing plants that
help each other (tomatoes + basil, the "three sisters" of beans + corn + squash)
and keeping apart pairs that fight (fennel near almost anything). This tab gives
you the consensus pairings for the plant you're looking at, and lets you peek at
each companion's needs or jump straight into its full care guide before deciding
what to grow alongside it.

### Every flow on this tab

#### 1. Read the pairings

- **Beneficial** and **Harmful** sections open by default; **Neutral** is collapsed.
- Each row shows the companion's name and a short reason it helps or hurts.

#### 2. Quick peek (ⓘ)

- Tap the **ⓘ** to expand a row: you get the reason plus the same info pills you
  see in plant search — sunlight, watering, edible, toxicity, wildlife — and a
  description. The data comes from our library first, then the wider plant
  databases, so common plants like tomato or basil fill in even if they aren't
  in our own library yet.

#### 3. Open the full care guide

- Tap a companion's **name** to open its full guide — Care, Grow Guide, its own
  Companions, and Light — without leaving the plant you started on.

#### 4. Add companions to your Shed

- Tick any companions, then **Add N to Shed**. Choose where each comes from
  (our database, AI, or Verdantly) and confirm — they're added with sensible
  watering/harvest reminders.

### Information on display — what every field means

- **Reason chip** — why the pairing is beneficial or harmful (pest deterrent, nitrogen fixing, pollinator attractor, allelopathy, nutrient competition…).
- **Pills** — sunlight need, watering frequency, edible, toxic-to-pets/humans, wildlife it attracts.
- **Description** — a short profile of the companion.

### Tier-by-tier experience

- **Free / Apprentice:** Verdantly-sourced companions work; AI generation is gated.
- **Botanist+:** AI companion generation unlocks (for non-Verdantly plants); higher rate limits at Sage/Evergreen.
- ⓘ peek and full care guides work for everyone (library + Verdantly are free).

### Common mistakes / pitfalls

- **Treating companion data as gospel.** It's traditional advice plus some science — results vary by climate and soil.
- **Antagonist caught between companions.** If X pairs with this plant and Y avoids it, but X and Y also clash, you'll have to compromise on layout.
- **Expecting every companion in our library.** Many aren't yet — the ⓘ falls back to the wider plant databases, and opening the full guide will generate one with AI only when nothing else is found.

### Recommended workflows

- **Pre-planting:** open the Companions tab for each plant in your plan, peek at the ones you don't know, and sketch a layout that respects most pairings.
- **Mid-season:** if pests appear, scan the beneficial list for known deterrents and add a couple to the Shed.

### What to do if something looks wrong

- **Empty tab:** no companion data was found for this species — try the plant's Verdantly/AI source, or check back later.
- **"Try again" banner:** the first call can fail on a cold start; tap retry.
- **Hit the limit:** AI companion generation is rate-limited per tier — wait a little and reopen (cached plants won't count against you).
